# SOURCES_AND_RISKS

## Data sources

Recommended sources:
- 2GIS Places API;
- Google Places API;
- Yandex Organization Search API;
- OpenStreetMap / Overpass as an auxiliary source.

## Data-source notes

Official APIs are preferred because they have documentation, rate limits and predictable response formats.
OpenStreetMap can be useful for geography, but business contact data is often incomplete.

## Risk policy

Use documented APIs and public business information only.
Avoid protected-page scraping, captcha bypassing and collection of irrelevant personal data.

For outreach, use manual targeted contact and ask permission before sending a detailed review.
Do not promise a guaranteed increase in sales or leads. Sell a concrete list of completed work.

## Platform risks

- API access can be paid or limited.
- API terms can restrict storing or reusing some fields.
- HTML scraping can cause blocks.
- OSM data can be incomplete.

## Rule

If a data source or method is questionable, check official documentation first and only then build.
